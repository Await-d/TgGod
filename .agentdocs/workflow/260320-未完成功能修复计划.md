## 实施方案

### 阶段一：稳定性修复（P0+P1）

**目标**：消除运行时崩溃风险，补全后端测试和 Jellyfin UI

#### T-01: 修复 complete_health_monitoring.py NotImplementedError

**文件**: `backend/app/services/complete_health_monitoring.py:154`

**方案**:
- 定位 `raise NotImplementedError` 所在方法
- 分析方法签名和上下文，实现最小可用逻辑（返回默认值或 pass）
- 确保服务正常启动不崩溃

#### T-02: 建立后端测试框架

**文件新增**:
```
backend/tests/
├── __init__.py
├── conftest.py              # pytest fixtures（DB、客户端）
├── test_rule_service.py     # 规则引擎测试
└── test_task_execution.py   # 任务调度基础测试
```

**方案**:
- conftest.py 使用 SQLite 内存数据库，隔离测试环境
- 每个服务最少覆盖 happy path + 一个错误路径

#### T-03: Jellyfin 前端配置 UI

**文件**: `frontend/src/pages/TaskManagement.tsx`

**方案**:
- 在创建/编辑任务表单中添加「Jellyfin 兼容模式」折叠面板
- 包含字段：`use_jellyfin_structure`（开关）
- 与现有 API 请求体对齐，后端已支持
