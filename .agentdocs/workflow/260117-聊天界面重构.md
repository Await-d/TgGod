## 任务概述
重构 ChatInterface 聊天界面，解决当前布局在不同端（桌面端、平板、移动端）展示不友好的问题，提供更好的用户体验和代码可维护性。

## 当前问题分析

### 1. 布局问题
- **复杂的响应式逻辑**: 当前使用多个状态和条件判断来处理不同屏幕尺寸
- **硬编码的断点**: 断点值分散在代码中，难以统一管理
- **布局切换不流畅**: 桌面端和移动端使用完全不同的布局结构
- **平板体验不佳**: 平板模式介于桌面和移动之间，体验不够优化

### 2. 代码结构问题
- **组件过大**: ChatInterface.tsx 超过1000行，难以维护
- **状态管理混乱**: 多个状态散落在组件中，缺乏统一管理
- **重复代码**: 桌面端和移动端有大量重复的逻辑
- **CSS 复杂**: ChatInterface.css 包含大量覆盖样式和 !important

### 3. 性能问题
- **不必要的重渲染**: 状态变化导致整个组件重新渲染
- **事件监听器过多**: resize、scroll 等事件监听器未优化
- **内存泄漏风险**: 部分 useEffect 清理不完整

### 4. 用户体验问题
- **移动端群组列表**: 抽屉式体验不够流畅
- **平板端布局**: 宽度调整不够智能
- **键盘弹起**: 虚拟键盘弹起时布局调整不够优雅
- **手势操作**: 手势冲突和响应不够灵敏

## 重构方案设计

### 方案1: 渐进式重构（推荐）⭐

#### 核心思路
采用渐进式重构策略，保持功能稳定的同时逐步优化代码结构和用户体验。

#### 设计原则
1. **移动优先**: 从移动端体验出发，向上适配桌面端
2. **组件化**: 将大组件拆分为小的、可复用的组件
3. **统一布局系统**: 使用统一的布局组件和断点系统
4. **性能优先**: 优化渲染性能和内存使用

#### 技术方案

##### 1. 布局系统重构

**新的布局架构**:
```
ChatInterface (容器)
├── ChatLayout (统一布局管理器)
│   ├── ChatSidebar (侧边栏 - 桌面/平板)
│   │   └── GroupList (群组列表)
│   ├── ChatMain (主内容区)
│   │   ├── ChatHeader (头部)
│   │   ├── MessageList (消息列表)
│   │   └── ChatFooter (底部输入区)
│   └── ChatDrawer (抽屉 - 移动端)
│       └── GroupList (群组列表)
```

**响应式断点统一管理**:
```typescript
// constants/breakpoints.ts
export const BREAKPOINTS = {
  mobile: 768,      // ≤768px
  tablet: 1024,     // 769-1024px
  desktop: 1025,    // ≥1025px
} as const;

export const LAYOUT_CONFIG = {
  mobile: {
    sidebarType: 'drawer',
    sidebarWidth: '80%',
    showSidebarByDefault: false,
  },
  tablet: {
    sidebarType: 'fixed',
    sidebarWidth: 280,
    showSidebarByDefault: true,
  },
  desktop: {
    sidebarType: 'resizable',
    sidebarWidth: 320,
    minWidth: 260,
    maxWidth: 400,
    showSidebarByDefault: true,
  },
};
```

##### 3. 状态管理优化

**使用 Context + Hooks 统一管理**:
```typescript
// contexts/ChatContext.tsx
interface ChatContextValue {
  // 布局状态
  layout: {
    deviceType: 'mobile' | 'tablet' | 'desktop';
    sidebarVisible: boolean;
    sidebarWidth: number;
    isKeyboardVisible: boolean;
  };

  // 聊天状态
  chat: {
    selectedGroup: TelegramGroup | null;
    messages: TelegramMessage[];
    filter
}

// 使用
const { layout, chat, actions } = useChatContext();
```

##### 4. CSS 架构重构

**采用 CSS Modules + CSS Variables**:
```css
/* ChatLayout.module.css */
.chatLayout {
  --sidebar-width: var(--chat-sidebar-width, 320px);
  --header-height: 60px;
  --footer-height: 80px;

  display: grid;
  grid-template-areas:
    "sidebar header"
    "sidebar main"
    "sidebar footer";
  grid-template-columns: var(--sidebar-width) 1fr;
  grid-template-rows: var(--header-height) 1fr var(--footer-height);
  height: 100%;
}

/* 移动端 */
@media (max-width: 768px) {
  .chatLayout {
    grid-template-areas:
      "header"
      "main"
      "footer";
    grid-template-columns: 1fr;
  }
}
```

##### 5. 性能优化

**优化策略**:
1. **虚拟滚动**: 使用 react-window 或 react-virtualized 优化消息列表
2. **懒加载**: 图片和媒体内容懒加载
3. **防抖节流**: resize、scroll 事件使用防抖节流
4. **React.memo**: 对列表项组件使用 memo 优化
5. **useCallback**: 缓存事件处理函数
6. **代码分割**: 按需加载大型组件
